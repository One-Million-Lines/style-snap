# Contributing

Thanks for your interest in SnapStyle.

## Local setup

```bash
npm install
npm run dev
```

## Before opening a PR

- Run `npm run build`
- Run `npm run lint`
- Keep copy and screenshots aligned with current demo behavior

## Guidelines

- Do not present simulated functionality as live AI generation.
- Keep template metadata in `src/data/templates.ts` consistent.
- If you add new flows, update the README and route list.
- Do not commit local `.env` files or personal test assets.

## PR checklist

- Explain the change
- Describe how you tested it
- Include screenshots for visible UI updates
